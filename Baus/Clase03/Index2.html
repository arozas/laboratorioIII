Las etiquetas y la diagramación de la pagina

Introducción.

Antiguamente para la diagramación y los esquemas de las páginas web se realizaban con las etiquetas div y span, que son etiquetas html contenedoras que no modifican la apariencia de la web ni son semanticas, por lo tanto su función era la de agrupar elementos internamente, aprovechando el Box Model. Div es una etiqueta en bloque y spam es una etiqueta inline, por lo tanto nos sirven cada una para agrupar elementos y texto respectivamente.

Su función real es la de agrupar estos elementos para luego ser apuntados por el script JavaScript o por lo hoja de estilo de CSS.

El problema del uso esta etiquetas es el sobreuso de estas etiquetas, lo que generaba un código de marcado o de maquetado ilegible, en su momento esto se solventaba con el uso de los IDs que nos permitia por medio de ese atributo identificar cada una de esas etiquetas.

Apartir de que aparece HTML 5, la web semantica y sobre todo el analisis de posicionamiento de los motores be busqueda, la forma de diagramar y agrupar los elementos cambió.

Etiqueta <section>.

El elemento <section> de HTML es una etiqueta que se utiliza para definir secciones de contenido en una página web. Se utiliza para agrupar contenido relacionado y facilitar la lectura y comprensión de la página.

La etiqueta <section> se puede utilizar en diferentes contextos y se puede anidar en otras secciones para crear una estructura jerárquica. El contenido de una sección puede incluir otros elementos HTML, como encabezados (<h1>-<h6>), párrafos (<p>), listas (<ul>, <ol>) e incluso otras secciones (<section>).

Una sección debe ser independiente y autónoma en términos de contenido. Es decir, debe poder ser leída y comprendida por sí sola, sin depender del contexto en el que se encuentra. Además, una sección debe tener un encabezado que describa el contenido que contiene. El encabezado puede ser un <h1>-<h6> o cualquier otro elemento de encabezado apropiado.

La etiqueta <section> es muy útil para estructurar el contenido de una página web, especialmente en páginas largas con muchos elementos. Ayuda a los usuarios a comprender la organización y jerarquía del contenido y puede mejorar la accesibilidad y la usabilidad de la página.

Ejemplo de uso: 


                        <section>
                            <h2>Introducción</h2>
                            <p>Este es un párrafo introductorio que describe el tema del artículo.</p>
                        </section>
                            
                        <section>
                            <h2>Desarrollo</h2>
                            <p>En esta sección se presentan los detalles del tema, divididos en subsecciones.</p>
                            <section>
                                <h3>Subsección 1</h3>
                                <p>Este es el contenido de la subsección 1.</p>
                             </section>
                            <section>
                                <h3>Subsección 2</h3>
                                <p>Este es el contenido de la subsección 2.</p>
                            </section>
                        </section>

                        <section>
                            <h2>Conclusión</h2>
                            <p>En esta sección se presentan las conclusiones del artículo.</p>
                        </section>
En el ejemplo anterior, se utiliza la etiqueta <section> para agrupar los diferentes elementos del artículo en tres secciones principales: Introducción, Desarrollo y Conclusión. Cada sección tiene un encabezado descriptivo y contiene el contenido relacionado. La sección de Desarrollo también contiene dos subsecciones anidadas.

Algunos de los beneficios de utilizar la etiqueta <section> son:
  1. Mejora la accesibilidad: Al utilizar una estructura de secciones clara y bien definida, se facilita la navegación y comprensión de la página para las personas que utilizan lectores de pantalla o tienen discapacidades visuales.
  2. Mejora el SEO: Al utilizar una estructura de secciones clara y bien definida, se mejora la capacidad de los motores de búsqueda para indexar y entender el contenido de la página.
  3. Mejora la usabilidad: Al utilizar una estructura de secciones clara y bien definida, se facilita la navegación y comprensión de la página para los usuarios.
  4. Facilita el mantenimiento y la actualización del contenido: Al dividir el contenido en secciones lógicas y autónomas, se facilita la actualización y el mantenimiento del contenido de la página.
Es importante destacar que el uso excesivo de la etiqueta <section> puede tener un efecto negativo en la accesibilidad y usabilidad de la página. Es recomendable utilizar la etiqueta con moderación y asegurarse de que cada sección sea clara, bien definida y autónoma. Además, es importante utilizar los elementos de encabezado adecuados para cada sección y asegurarse de que sean jerárquicamente coherentes.

Etiqueta <header>>.

El elemento <header> es un elemento semántico en HTML que se utiliza para definir el encabezado o cabecera de una página o sección de una página web. El contenido que se encuentra dentro de la etiqueta <header> suele ser contenido introductorio o informativo, como el título principal de la página, el logotipo de la marca, la información de contacto, el menú de navegación principal, etc.

El elemento <header> se utiliza para proporcionar una estructura semántica clara y coherente para el contenido de la cabecera de una página o sección de una página web. Esto facilita la navegación y comprensión del contenido de la página para los usuarios y los motores de búsqueda.

El elemento <header> puede contener cualquier tipo de contenido HTML, como encabezados, párrafos, imágenes, listas, enlaces, etc. También se puede utilizar junto con otros elementos semánticos como <nav>, <h1>, <h2>, <h3>, etc. para proporcionar una estructura semántica clara y coherente.

Es importante destacar que el uso adecuado del elemento <header> puede mejorar la accesibilidad y la usabilidad de una página web. Al proporcionar una estructura semántica clara y coherente para el contenido de la cabecera de la página, se facilita la navegación y comprensión del contenido para las personas que utilizan lectores de pantalla o tienen discapacidades visuales. Además, el uso adecuado del elemento <header> también puede mejorar la optimización para motores de búsqueda al indicar claramente qué parte del contenido es de cabecera y qué parte es de contenido principal.

Etiqueta <nav>.

El elemento <nav> es un elemento semántico en HTML que se utiliza para definir un bloque de enlaces de navegación. Este elemento se utiliza para agrupar enlaces que permiten a los usuarios navegar por diferentes secciones de un sitio web, como por ejemplo enlaces a páginas de inicio, acerca de nosotros, productos, servicios, contacto, etc.

El elemento <nav> se debe utilizar cuando el bloque de enlaces de navegación es parte del contenido principal de la página, y no se debe utilizar para enlaces secundarios o de navegación externa que no estén relacionados con el contenido de la página actual.

El elemento <nav> puede contener cualquier tipo de enlace, incluyendo enlaces a otras páginas web, secciones del sitio web, anclas internas en la misma página, etc. También puede contener otros elementos HTML como encabezados, párrafos, listas, imágenes, etc. para proporcionar más contexto sobre los enlaces de navegación.

Es importante destacar que el elemento <nav> no proporciona ningún estilo o diseño por sí mismo, por lo que es necesario aplicar estilos CSS para dar formato al bloque de enlaces de navegación.

El uso adecuado del elemento <nav> puede mejorar la accesibilidad y la usabilidad de un sitio web al proporcionar una estructura semántica clara y coherente para la navegación. Además, también puede mejorar la optimización para motores de búsqueda al indicar claramente qué parte del contenido es de navegación y qué parte es de contenido principal.

Etiqueta <main>.

El elemento <main> es un elemento semántico en HTML que se utiliza para indicar el contenido principal o central de una página web. Este elemento se utiliza para separar el contenido principal del contenido secundario, como la cabecera, el pie de página, las barras laterales y otros elementos de navegación.

El elemento <main> se utiliza para proporcionar una estructura semántica clara y coherente para el contenido principal de una página web. Este elemento se utiliza generalmente una sola vez en una página web y contiene el contenido principal de la página. El contenido que se encuentra dentro de la etiqueta <main> suele ser el contenido que se muestra primero al cargar la página, como los artículos de un blog, el contenido principal de una página de productos o servicios, etc.

El uso adecuado del elemento <main> es importante para la accesibilidad y la usabilidad de una página web. Al proporcionar una estructura semántica clara y coherente para el contenido principal de una página, se facilita la navegación y comprensión del contenido para los usuarios y los motores de búsqueda. Además, el uso adecuado del elemento <main> también puede mejorar la accesibilidad para las personas que utilizan lectores de pantalla o tienen discapacidades visuales.

Es importante destacar que el elemento <main> no debe utilizarse para separar el contenido secundario o el contenido adicional de una página. En su lugar, se deben utilizar elementos semánticos adecuados como <header>, <footer>, <nav>, <aside> y otros elementos para separar y definir claramente el contenido secundario y adicional.

Etiqueta <article>>.

El elemento <article> en HTML se utiliza para representar una sección autónoma y completa de contenido que puede ser distribuida, reutilizada o citada de forma independiente del resto de la página web. Es un elemento semántico que indica que el contenido dentro de él es independiente del resto del contenido de la página y puede ser considerado como una entidad completa en sí mismo.

El elemento <article> se utiliza para encapsular un contenido que tiene una idea o tema principal. Se puede utilizar para representar artículos de noticias, entradas de blog, reseñas de productos, publicaciones de redes sociales y cualquier otro tipo de contenido que pueda ser distribuido de forma independiente.

Es importante tener en cuenta que el elemento <article> no debe confundirse con el elemento <section>. Mientras que el elemento <section> se utiliza para agrupar contenido relacionado temáticamente, el elemento <article> se utiliza para representar una pieza completa e independiente de contenido.

El elemento <article> se puede utilizar dentro de cualquier elemento de sección de HTML, como <body>, <section>, <aside>, <nav>, entre otros. Es común utilizar el elemento <header> y <footer> dentro del elemento <article> para proporcionar información adicional, como el título del artículo y la fecha de publicación.

En resumen, el elemento <article> es un elemento semántico en HTML que se utiliza para representar una sección autónoma y completa de contenido que puede ser distribuida, reutilizada o citada de forma independiente del resto de la página web. Este elemento se utiliza para encapsular contenido que tiene una idea o tema principal y puede ser utilizado dentro de cualquier elemento de sección de HTML.

Etiqueta <aside>.

El elemento <aside> en HTML se utiliza para representar contenido relacionado pero secundario al contenido principal de una página web. Es un elemento semántico que indica que el contenido dentro de él está relacionado con el contenido principal, pero no es esencial para entenderlo.

El elemento <aside> se puede utilizar para una amplia variedad de contenidos relacionados, como publicidad, enlaces relacionados, biografías, información adicional, redes sociales, entre otros. El contenido dentro del elemento <aside> puede ser presentado de diferentes maneras, como texto, imágenes, video, audio o cualquier otro tipo de contenido.

Es importante tener en cuenta que el elemento <aside> no debe utilizarse para el contenido principal de una página web. En lugar de eso, debe utilizarse para contenido relacionado secundario que no sea esencial para la comprensión del contenido principal.

El elemento <aside> se puede utilizar dentro de cualquier elemento de sección de HTML, como <body>, <section>, <article>, entre otros. Es común utilizarlo dentro del elemento <main> para proporcionar información adicional relacionada con el contenido principal de la página.

En resumen, el elemento <aside> es un elemento semántico en HTML que se utiliza para representar contenido relacionado pero secundario al contenido principal de una página web. Este elemento se puede utilizar para una amplia variedad de contenidos relacionados y se utiliza dentro de cualquier elemento de sección de HTML. Es importante tener en cuenta que el elemento

no debe utilizarse para el contenido principal de una página web y debe utilizarse para contenido secundario relacionado que no sea esencial para la comprensión del contenido principal.

Etiqueta <footer>.

El elemento <footer> en HTML se utiliza para representar la sección de pie de página de un documento o sección. Es un elemento semántico que indica el final de una sección o de un documento y contiene información que generalmente se considera menos importante que el contenido principal.

El elemento <footer> se utiliza para proporcionar información adicional sobre la página web, como los datos de contacto, la información de derechos de autor, los enlaces a la política de privacidad, los términos y condiciones, entre otros. También puede contener enlaces a las redes sociales, el mapa del sitio y otros elementos de navegación que no se consideran parte del contenido principal de la página.

Es importante tener en cuenta que el elemento <footer> no debe confundirse con el elemento <bottom> o la etiqueta de cierre </html>. El elemento <footer> se utiliza para representar la sección de pie de página, mientras que el elemento <bottom> se utiliza para representar el final del contenido de la página y puede contener cualquier contenido adicional necesario, como los scripts de JavaScript y las hojas de estilo.

El elemento <footer> se puede utilizar dentro de cualquier elemento de sección de HTML, como <body>, <article>, <section>, <aside>, <nav>, entre otros. En general, se recomienda colocar el elemento

al final de la sección o del documento para que los usuarios puedan acceder fácilmente a la información adicional después de leer el contenido principal.

En resumen, el elemento <footer> es un elemento semántico en HTML que se utiliza para representar la sección de pie de página de un documento o sección. Este elemento se utiliza para proporcionar información adicional sobre la página web y se recomienda colocarlo al final de la sección o del documento.
